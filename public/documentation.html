My Flix
Application of the Site

The site will be able to allow users to complete a few steps whilst they are enjoying the site. These include:
• Signing up to the Site,
• Editing personal data incuding favourite movies,
• Deleting their accounts,
• Searching for movies and additional information that they require from the substancial data base.

Description of Endpoint Endpoint URL Method Type Format of Data in Request Format of Data in Response
Search for all movies in the database http://localhost:8080/movies GET None The server will return a detailed list of all movies in the database
User Registration http://localhost:8080/users/ POST The User will add the following Json code:
{ name: "Richard Garret",
favoriteMovies: ["War Dogs"] }
 		The server will store the information and return a response as follows:
You will notice that a unique id has been added to the new user.

"favoriteMovies":["War Dogs"] },
"id": "5e4d3669-aee6-4aa4-96ca-d78750ae2912"
Editting username. /users/:id PUT The User will add the following Json code:
{ "name": "Richard Warrick Garret",
"favoriteMovies": ["War Dogs"] }
The server will change the name of the user and then additionally produce the following as proof of change:
{ "id": 2,
"name": "Richard Warrick Garret",
"favoriteMovies": ["War Dogs"] }
Adding favorite movies. http://localhost:8080/users/:id/:movieTitle POST http://localhost:8080/users/1/Avatar The server will add information and then return the following statement:
Avatar has been added to user 1's array
Removing favorite movies. http://localhost:8080/users/:id/:movieTitle DELETE http://localhost:8080/users/1/Avatar The server will add information and the then return the following statement:
Avatar has been removed from user 1's array
Removing account. http://localhost:8080/users/:id DELETE http://localhost:8080/users/1 The server will delete the account and return the following confirmation:
User 1 account has been closed

Status Codes: Description of the Status Code used:
Code 200 200 OK
The request succeeded. The result meaning of "success" depends on the HTTP method:
GET: The resource has been fetched and transmitted in the message body.
HEAD: The representation headers are included in the response without any message body.
PUT or POST: The resource describing the result of the action is transmitted in the message body.
Code 201 201 Created
The request succeeded, and a new resource was created as a result.
This is typically the response sent after POST requests, or some PUT requests.
Code 400 400 Bad Request
The server cannot or will not process the request due to something that is perceived to be a client error (e.g., malformed request syntax, invalid request message framing, or deceptive request routing).
